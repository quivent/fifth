App from Spec

Generate complete, runnable applications from YAML specifications. Multi-stage LLM synthesis: architecture → schema → API → frontend → tests. Validation at each stage.

Run

fifth examples/app-from-spec/main.fs generate spec.yaml output/full generation
fifth examples/app-from-spec/main.fs stage schema spec.yamlsingle stage

The Pipeline

spec.yaml
    ↓
Stage 1: Architecture  →  architecture.md (components, data flow)
    ↓
Stage 2: Schema        →  schema.sql, migrations/
    ↓
Stage 3: API           →  routes/, handlers/, middleware/
    ↓
Stage 4: Frontend      →  components/, pages/, services/
    ↓
Stage 5: Tests         →  tests/unit/, tests/integration/

Stage Details

  1. Architecture — System components, data flow, interfaces. Validated for coherence.
  2. Schema — SQL CREATE statements with foreign keys, indexes. Syntax-checked via sqlite3 :memory:
  3. API — REST routes with CRUD, auth middleware, validation. Route coverage verified.
  4. Frontend — React components, forms, API client hooks. Field mapping checked.
  5. Tests — Unit, integration, e2e tests. Coverage for each source file.

Validation & Retry

Each stage validates output before proceeding. On failure, feedback is included in regeneration prompt:

Previous generation had issues:
- Missing foreign key for Task.user_id
- Invalid SQL syntax on line 23
Please regenerate fixing these issues.

Supported Stacks

  • Backend — Express, FastAPI, Go/Chi
  • Database — SQLite, PostgreSQL
  • Frontend — React, static HTML

Example Spec

name: TaskTracker
entities:
  - name: Task
    fields:
      - {name: title, type: string, required: true}
      - {name: status, type: enum, values: [todo, done]}
  - name: User
    fields:
      - {name: email, type: string, unique: true}
relations:
  - {type: belongs_to, from: Task, to: User}
features: [authentication, crud, filtering]
stack: {backend: express, database: sqlite, frontend: react}

Why Fifth?

  • Shell-out orchestration — curl for LLM, jq/yq for parsing, mkdir for scaffolding
  • Linear execution — Clear stage-by-stage flow with checkpoints
  • Validation hooks — Each stage can validate and retry
