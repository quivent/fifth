Codebase Indexer

Production-grade semantic code indexing for AI agents. Embed and search codebases using vector similarity — find conceptually related code that grep can't.

Run

fifth examples/codebase-indexer/main.fs index .index current directory
fifth examples/codebase-indexer/main.fs query "authentication handler"semantic search

Architecture

Source Files    →    Code Chunker    →    Embedding API    →    SQLite Vector Store
 *.js *.py *.fs       (by function)       (OpenAI/Ollama)        (vectors + metadata)
                                                                        ↓
                                          Semantic Search    ←    Cosine Similarity

Why Semantic Search?

Traditional text search fails for code. Query "authentication handler" — grep misses verify_user(), but semantic search finds it.

Context RetrievalFeed relevant code to LLMs without token overflow
Code NavigationFind all code related to a concept
Implicit DependenciesDiscover relationships grep can't find

Intelligent Chunking

Preserves semantic coherence — splits by function/class boundaries, not arbitrary line counts. Detects function, def, : (Forth words), class, struct.

Embedding Models

  • text-embedding-ada-002 — OpenAI API, 1536 dims, excellent quality
  • nomic-embed-text — Ollama local, 768 dims, good quality
  • all-minilm — Ollama local, 384 dims, very fast

Vector Storage

SQLite stores everything — zero deployment, single file, 100K vectors with sub-second search. Vectors as JSON arrays with file path, line numbers, chunk type metadata.

Why Fifth?

  • Shell composition — curl for APIs, sqlite3 for storage, no dependencies
  • Dual buffer system — Build SQL commands while escaping content without allocation
  • Composable pipeline — read-file-chunks → embed-chunk → store-embedding

Commands

  • index <path> — Index source files
  • query <text> [limit] — Semantic search
  • context <text> — Get chunks formatted for LLM prompts
  • stats — Show index statistics
