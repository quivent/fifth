Training Data Generator

Extract high-quality training data from codebases for fine-tuning code models. Docstring pairs, commit diffs, test mappings — with deduplication and quality filters.

Run

fifth examples/training-data-gen/main.fs /path/to/repoextract training data
fifth examples/training-data-gen/main.fs . --format=sharegptShareGPT format

Extraction Strategies

Docstring → CodeFunction docstrings as instructions, bodies as outputs
Commit → DiffGit commit messages paired with code changes
Test → ImplementationTest assertions as specs, implementations as solutions

Quality Filters

Not all samples are useful. The generator filters:

  • Too short — Reject trivial pass-only functions
  • Incomplete — Skip TODO/FIXME docstrings
  • Merge commits — Automated, uninformative
  • Huge diffs — >500 lines too coarse-grained

Deduplication

Hash-based exact dedup plus near-duplicate detection. "Calculate the sum" and "Compute the addition" are semantically equivalent — keep only one.

Output Formats

  • Alpaca — {"instruction": "...", "output": "..."}
  • ShareGPT — {"conversations": [...]}
  • Completion — {"text": "..."} for pre-training

Why Fifth?

  • Natural text pipeline — Buffer system maps to text transformation
  • Shell-out for git — Parse history without git libraries
  • No allocation overhead — Static buffers for millions of samples
  • Composable filters — min-length-ok? has-docstring? no-todo? and and

Integration

JSONL output works directly with Hugging Face datasets, axolotl, LitGPT, or stream line-by-line to custom trainers.
